Future 对象

Future 对象可以简单的理解为一个异步操作的占位符，Future 对象有以下几个特性：

  1. 包含了很多属性，包括 _result 和 _callbacks，这两个属性分别存储异步操作的结果及回调函数。
  2. 包含了很多方法，比如添加回调函数，设置异步操作结果等。
  3. 当 Future 对象完成对应的异步操作后，该对象会被 set_done，然后遍历运行 _callbacks 中的回调函数。

class Future(object):

    def __init__(self):
        self._done = False      # 是否执行结束
        self._result = None     # 执行结果
        self._callbacks = []    # 保存回调函数

    def result(self, timeout=None):
        """
        成功返回结果，失败抛出异常
        """
        self._clear_tb_log()
        if self._result is not None:
            return self._result
        if self._exc_info is not None:
            try:
                raise_exc_info(self._exc_info)
            finally:
                self = None
        self._check_done()
        return self._result

    def add_done_callback(self, fn):
        """
        给 Future 对象添加指定的回调函数
        """
        if self._done:
            fn(self)
        else:
            self._callbacks.append(fn)

    def set_result(self, result):
        """
        保存结果，并执行结束后操作
        """
        self._result = result
        self._set_done()

    def _set_done(self):
        """
        标识执行结束并运行回调函数
        """
        self._done = True
        for cb in self._callbacks:
            try:
                cb(self)
            except Exception:
                app_log.exception('Exception in callback %r for %r', cb, self)
        self._callbacks = None

def coroutine(func, replace_callback=True):
    return _make_coroutine_wrapper(func, replace_callback=True)
    

def _make_coroutine_wrapper(func, replace_callback):

    wrapped = func
    if hasattr(types, 'coroutine'):
        func = types.coroutine(func)

    @functools.wraps(wrapped)
    def wrapper(*args, **kwargs):
        """
        简化版代码：
        future = TracebackFuture()
        result = func(*args, **kwargs)
        if isinstance(result, GeneratorType):
            yielded = next(result)
            _futures_to_runners[future] = Runner(result, future, yielded)
            yielded = None
        return future
        """
        future = TracebackFuture()

        if replace_callback and 'callback' in kwargs:
            callback = kwargs.pop('callback')
            IOLoop.current().add_future(
                future, lambda future: callback(future.result()))

        try:
            result = func(*args, **kwargs)
        except (Return, StopIteration) as e:
            result = _value_from_stopiteration(e)
        except Exception:
            future.set_exc_info(sys.exc_info())
            return future
        else:
            if isinstance(result, GeneratorType):
                try:
                    orig_stack_contexts = stack_context._state.contexts
                    yielded = next(result)
                    if stack_context._state.contexts is not orig_stack_contexts:
                        yielded = TracebackFuture()
                        yielded.set_exception(
                            stack_context.StackContextInconsistentError(
                                'stack_context inconsistency (probably caused '
                                'by yield within a "with StackContext" block)'))
                except (StopIteration, Return) as e:
                    future.set_result(_value_from_stopiteration(e))
                except Exception:
                    future.set_exc_info(sys.exc_info())
                else:
                    _futures_to_runners[future] = Runner(result, future, yielded)
                yielded = None
                try:
                    return future
                finally:
                    future = None
        future.set_result(result)
        return future

    wrapper.__wrapped__ = wrapped
    wrapper.__tornado_coroutine__ = True
    return wrapper

大致过程如下所示：

  1. 生成一个 Future 对象
  2. 运行被装饰函数并将结果赋值给 result。tornado 的“异步”实现基于 generator，所以一般情况下 result 是一个 generator 对象。
  3. yielded = next(result) 执行到被装饰函数的第一次 yield 并将结果赋值给 yielded。一般情况下 yielded 是一个 Future 对象。
  4. Runner(result, future, yielded)。
  5. return future 返回异步操作对象。

这里需要解析一下第4步中的 Runner() 类都做了些什么。

Runner() 类

Runner() 可以自动的将异步操作结果 send() 至生成器中止的地方。tornado 的协程是基于 generator 实现的，多数情况下存在 generator 的嵌套，比如经常会 yield 一个 generator，当 A 生成器 yield B 生成器时，会执行以下两步：

  1. 中止 A 的执行转而执行 B
  2. 当 B 执行完成后，将 B 的执行结果 send() 至 A 中止的地方，恢复执行 A。

而上面这两步过程就是 Runner() 的主要功能：控制生成器的执行与中止，并适时的将 B 的执行结果 send() 至 A ，从而唤醒 A 生成器继续执行。

class Runner(object):

    def __init__(self, gen, result_future, first_yielded):
        ...
        if self.handle_yield(first_yielded):
            gen = result_future = first_yielded = None
            self.run()
            
    def run(self):
        """Starts or resumes the generator, running until it reaches a
        yield point that is not ready.
        """
        if self.running or self.finished:
            return
        try:
            self.running = True
            while True:
                future = self.future
                if not future.done():
                    return
                self.future = None  # 清空 self.future
                try:
                    value = future.result() # 获取 future 对象的结果
                    yielded = self.gen.send(value) # send 结果，并将 self.gen 返回的值赋值给 yielded（一般情况下这也是个future对象）
                except (StopIteration, Return) as e:
                    self.finished = True
                    self.future = _null_future
                    if self.pending_callbacks and not self.had_exception:
                        raise LeakedCallbackError(
                            "finished without waiting for callbacks %r" %
                            self.pending_callbacks)
                    self.result_future.set_result(_value_from_stopiteration(e))
                    self.result_future = None
                    self._deactivate_stack_context()
                    return
                except Exception:
                    self.finished = True
                    self.future = _null_future
                    self.result_future.set_exc_info(sys.exc_info())
                    self.result_future = None
                    self._deactivate_stack_context()
                    return
                # 判断 yielded 对象是否执行完成，若已完成则直接返回，否则继续循环
                if not self.handle_yield(yielded):
                    return
                yielded = None
        finally:
            self.running = False

    def handle_yield(self, yielded):
        """
        处理 yield 返回对象
        """
        if _contains_yieldpoint(yielded):
            yielded = multi(yielded)

        if isinstance(yielded, YieldPoint):
            self.future = TracebackFuture()  # 一个刚刚初始化的Future对象
            # 省略 YieldPoint 相关处理
        else:
            # yielded 是一个 Future 对象（最常见的情况）
            try:
                self.future = convert_yielded(yielded)  # 解析 yielded 对象
            except BadYieldError:
                self.future = TracebackFuture()
                self.future.set_exc_info(sys.exc_info())
        
        # 判断 future 对象是否已完成，若没有则添加回调函数，该回调函数会执行 self.run()
        if not self.future.done() or self.future is moment:
            def inner(f):
                # Break a reference cycle to speed GC.
                f = None # noqa
                self.run()
            self.io_loop.add_future(
                self.future, inner)
        # 返回 future 对象是否被 done
            return False
        return True

总结

  1. Tornado 的协程是基于 generator 实现的，generator 可以通过 yield 关键字暂停执行，也可以通过 next() 或 send() 恢复执行，且 send() 可以直接向 generator 传递值。
  2. Future 对象负责将协程串联起来，每一个 Future 对象对应一个异步操作，可以为该对象添加许多回调函数，当异步操作完成后通过对 Future 对象进行 set_done 或者 set_result 就可以执行相关的回调函数。
  3. Runner() 负责实际的运行控制，它不断的将 generator yield 生成的 future 对象结果 send() 至 generator，当 generator 运行结束后，它会进行最后的包装工作，并对该 generator 对应的 Future 对象执行 set_result 操作。
